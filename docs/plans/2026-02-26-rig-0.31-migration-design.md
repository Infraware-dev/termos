# Rig 0.28 to 0.31 Migration Design

## Context

Upgrade rig-core from 0.28 to 0.31. Cargo.toml already bumped; code needs updating.

## Breaking Changes (Affecting This Codebase)

### 1. PromptHook trait reworked

- `CancelSignal` removed from all hook methods
- `on_tool_call` returns `ToolCallHookAction` (Continue/Skip/Terminate) instead of `()`
- `on_completion_response` returns `HookAction` (Continue/Terminate) instead of `()`
- New `internal_call_id: &str` parameter added to `on_tool_call` and `on_tool_result`
- New streaming hooks added (unused for now): `on_text_delta`, `on_tool_call_delta`

### 2. multi_turn renamed to max_turns

- `.multi_turn(N)` -> `.max_turns(N)` on PromptRequest

## Decision: ToolCallHookAction::Skip for HITL

When intercepting tool calls for user approval, use `ToolCallHookAction::Skip` with a reason string.
This ends the tool call gracefully and the reason is available as context.

## Files to Change

### orchestrator.rs

1. Imports: `CancelSignal` -> `HookAction`, `ToolCallHookAction`
2. `HitlHook::on_tool_call()`: remove `cancel_sig`, add `_internal_call_id`, return `ToolCallHookAction::Skip`
3. `HitlHook::on_completion_response()`: remove `cancel_sig`, return `HookAction::cont()`
4. `.multi_turn(1)` -> `.max_turns(1)` (2 sites)

### incident/mod.rs

1. `.multi_turn(1)` -> `.max_turns(1)` (if present)

### Unchanged

- All Tool implementations (trait unchanged)
- Agent builder calls
- Message types
- Client creation
