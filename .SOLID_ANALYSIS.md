- **File**: `src/terminal/state.rs` + nuovo `src/terminal/buffers.rs`
- **Problema Risolto**: God Object con 6 responsabilità diverse
- **Soluzione Implementata**:
  ```rust

      pub output_buffer: Vec<String>,

      pub cursor_position: usize,
      pub command_history: Vec<String>,
      pub history_position: Option<usize>,
      pub scroll_position: usize,
      pub mode: TerminalMode,
  }

  pub struct OutputBuffer { /* gestione output + scroll */ }

      pub input: InputBuffer,

      pub mode: TerminalMode,
  }
  ```

  - ❌ Codice che serve solo output dipende da cursor/history/scroll
  - ❌ Difficile da testare

  ```rust
      // ...
  }
  ```
  - ✅ Ogni componente con interfaccia focalizzata
  - ✅ Testabile indipendentemente
  - ✅ Dipendenze ridotte

  - Interfacce ridotte da 13 metodi su 1 classe a 3 interfacce separate

  ```rust
  pub struct DockerExecutor { /* ... */ }   // Future extension
  pub struct MockExecutor { /* ... */ }     // For testing
  ```

```rust
// Codice client doveva sapere tutto
state.output_buffer.push("line".to_string());
state.scroll_position = state.output_buffer.len() - 1;

state.input_buffer.insert(state.cursor_position, 'c');
state.cursor_position += 1;

let cmd = state.command_history[state.history_position.unwrap()].clone();
```

**Dopo** (Encapsulation):
```rust
// API pulita e focalizzata
state.add_output("line".to_string());

state.insert_char('c');

state.history_previous();  // Gestisce tutto internamente
```

## 📝 Note e Decisioni

### Decisioni Architetturali

#### 2025-11-08: TerminalState Split
- **Decisione**: Dividere in OutputBuffer, InputBuffer, CommandHistory
- **Rationale**:
  - God Object con 6 responsabilità diverse
  - Violazione SRP e ISP
  - Difficile da testare
- **Alternative Considerate**:
  1. ❌ Mantenere monolitico: non risolve problemi
  2. ❌ Dividere in 2 componenti (IO + History): insufficiente
  3. ✅ **Dividere in 3 componenti**: bilanciamento ottimale
- **Trade-offs**:
  - ➕ Migliore testabilità
  - ➕ Migliore encapsulation
  - ➕ Codice più chiaro
  - ➖ Leggermente più verboso (accettabile)
  - ➖ Indirezione aggiuntiva (minimale)

#### 2025-11-08: Cursor Position Strategy
- **Decisione**: `cursor_position` conta caratteri, non byte
- **Rationale**:
  - User mental model: caratteri, non byte
  - Compatibile con emoji/CJK
  - API più intuitiva
- **Alternative Considerate**:
  1. ❌ Byte indices: confuso per user, errori UTF-8
  2. ❌ Grapheme clusters: overkill per M1
  3. ✅ **Char indices**: bilanciamento corretto
- **Implementazione**: Helper `char_to_byte_idx()` per conversione

#### 2025-11-08: Backward Compatibility
- **Decisione**: Mantenere API TerminalState identica
- **Rationale**:
  - Minimizzare impatto su codice esistente
  - Refactoring incrementale
  - Facade pattern naturale
- **Risultato**: Zero breaking changes

### Best Practices Identificate

1. **Defensive Programming**
   - ✅ `debug_assert!` per invarianti in development
   - ✅ Clamping per safety in production
   - ✅ Saturating math per prevenire overflow

2. **UTF-8 Handling**
   - ✅ Sempre usare `chars()` per iterazione
   - ✅ `char_indices()` per posizioni
   - ✅ Mai assumere 1 char = 1 byte

3. **Testing**
   - ✅ Test edge cases (empty, boundary)
   - ✅ Test Unicode (emoji, CJK)
   - ✅ Test invariants (scroll after pop)

4. **Documentation**
   - ✅ Module-level docs con esempi
   - ✅ Doc comments su metodi pubblici
   - ✅ Inline comments per logica complessa

### Problemi Incontrati e Soluzioni

#### Problema 1: UTF-8 Test Failure
- **Sintomo**: `test_input_buffer_unicode` panic con "assertion failed: self.is_char_boundary(idx)"
- **Causa**: `cursor_position` usato come byte index in `String::insert()`
- **Soluzione**: Implementato `char_to_byte_idx()` per conversione
- **Lezione**: Sempre testare con Unicode, non solo ASCII

#### Problema 2: InputBuffer::take() Performance
- **Sintomo**: Code review identifica clone non necessario
- **Causa**: `self.buffer.clone()` invece di move
- **Soluzione**: `std::mem::take(&mut self.buffer)`
- **Impatto**: Eliminata allocazione per ogni input submit

#### Problema 3: Module Visibility
- **Sintomo**: Clippy warning "unused imports"
- **Causa**: `pub use buffers::{...}` ma non usato esternamente
- **Soluzione**: `mod buffers` privato invece di `pub mod`
- **Lezione**: Esportare solo ciò che è necessario pubblicamente
